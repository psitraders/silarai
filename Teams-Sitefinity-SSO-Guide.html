The Problem

When embedding a Sitefinity Renderer page inside Microsoft Teams, authentication breaks due to how Teams renders tab content:

  • Teams tabs run inside an iframe with a restricted webview
  • Standard OpenID Connect / OAuth redirect flows break inside iframes
  • Popup windows are blocked or lose Teams context
  • Third-party cookies are blocked (affects session persistence)
  • login.microsoftonline.com has iframe-embedding restrictions (X-Frame-Options)
🚫
The normal OpenID Connect login flow simply does not work inside Teams tabs. You need a purpose-built Teams SSO integration.
2

The Solution — Teams SSO + Token Exchange

Use Teams' native SSO to silently get an Azure AD token, then exchange it for a Sitefinity session cookie on the backend.

1User opens Teams Tab
↓
2/teams-tab page loads — no auth required
↓
3Teams SDK getAuthToken() — silent SSO, no popup, Azure AD issues token automatically
↓
4POST token to /auth/teams-exchange
↓
5On-Behalf-Of flow → Microsoft Graph /me → get email + name
↓
6HttpContext.SignInAsync() — sets auth cookie
↓
7Redirect to real Sitefinity Renderer page — fully authenticated
↓
8Sitefinity personalization works normally
✅
User experience: The tab loads and the user is logged in automatically — zero login prompts, zero redirects.

Azure AD App Registration (One-Time Setup)

Step 1 — Create the App Registration

  1. Go to portal.azure.com → Azure Active Directory → App Registrations
  2. Click New Registration
    • Name: SitefinityTeamsApp
    • Supported account types: Single tenant (your org)
    • Redirect URI: https://yourdomain.com/auth/teams-callback
  3. Click Register

Step 2 — Configure Authentication

  1. Go to Authentication tab
  2. Click Add a platform → Single Page Application
  3. Add Redirect URI: https://yourdomain.com/auth/teams-callback
  4. Under Implicit grant, check both: Access tokens and ID tokens
  5. Click Save

Step 3 — Expose an API

  1. Go to Expose an API tab
  2. Set Application ID URI: api://yourdomain.com/YOUR_CLIENT_ID
  3. Click Add a scope:
    • Scope name: access_as_user
    • Who can consent: Admins and users
    • Display name: Access Sitefinity as user
  4. Under Authorized client applications, add both Teams client IDs: 
    1fec8e78-bce4-4aaf-ab1b-5451cc387264   (Teams desktop & mobile)
5e3ce6c0-2b1f-4285-8d4b-75ee78787346   (Teams web)

Step 4 — API Permissions

  1. Go to API Permissions tab
  2. Click Add a permission → Microsoft Graph → Delegated
  3. Select User.Read
  4. Click Grant admin consent for your organization

Step 5 — Create Client Secret

  1. Go to Certificates & Secrets tab
  2. Click New client secret
  3. Set expiry (recommend 24 months)
  4. Copy the VALUE immediately — you only see it once!

Step 6 — Note Down These Values

ValueWhere to Find It
Tenant IDApp Registration → Overview page
Client IDApp Registration → Overview page
Client SecretCertificates & Secrets (copy immediately after creation)
⚠️
The Authorized client app IDs in Expose an API are mandatory. Without them, Teams SSO silent token will fail with a consent error.

Program.cs Configuration

Program.cs
var builder = WebApplication.CreateBuilder(args);

// ── Cookie Auth ────────────────────────────────────────────────────────────
builder.Services.AddAuthentication(CookieAuthenticationDefaults.AuthenticationScheme)
    .AddCookie(options =>
    {
        options.Cookie.Name = "SitefinityTeamsAuth";
        options.Cookie.SameSite = SameSiteMode.None;     // REQUIRED for Teams iframe
        options.Cookie.SecurePolicy = CookieSecurePolicy.Always;
        options.Cookie.HttpOnly = true;
        options.LoginPath = "/teams-tab";                // Teams tab IS the login page
        options.ExpireTimeSpan = TimeSpan.FromHours(8);
    });

builder.Services.Configure<CookiePolicyOptions>(options =>
{
    options.MinimumSameSitePolicy = SameSiteMode.None;
    options.Secure = CookieSecurePolicy.Always;
});

builder.Services.AddHttpClient();
builder.Services.AddControllers();

// ── Your existing Sitefinity Renderer services stay here ──────────────────

var app = builder.Build();

// ── iframe / CSP headers — MUST be registered FIRST ──────────────────────
app.Use(async (context, next) =>
{
    context.Response.Headers.Remove("X-Frame-Options");
    context.Response.Headers["Content-Security-Policy"] =
        "frame-ancestors 'self' https://teams.microsoft.com " +
        "https://*.teams.microsoft.com https://*.skype.com";
    await next();
});

app.UseCookiePolicy();
app.UseAuthentication();
app.UseAuthorization();
app.MapControllers();

// ── Your existing Sitefinity Renderer middleware stays here ───────────────

app.Run();
ℹ️
The iframe header middleware must come before all other middleware so the headers are set on every response, including Sitefinity Renderer pages.

        microsoftTeams.app.initialize().then(() => {
            microsoftTeams.authentication.getAuthToken({
                successCallback: function (token) {
                    // Send token to backend for exchange
                    fetch('/auth/teams-exchange', {
                        method: 'POST',
                        headers: { 'Content-Type': 'application/json' },
                        body: JSON.stringify({ token: token }),
                        credentials: 'include'   // important: sends/receives cookies
                    })
                    .then(r => r.json())
                    .then(data => {
                        if (data.success) {
                            window.location.href = data.redirectUrl;
                        } else {
                            document.querySelector('.msg').textContent =
                                'Login failed: ' + data.error;
                        }
                    });
                },
                failureCallback: function (error) {
                    // Fallback: trigger Teams-managed popup login
                    microsoftTeams.authentication.authenticate({
                        url: window.location.origin + '/auth/teams-popup',
                        width: 600,
                        height: 535,
                        successCallback: () => window.location.href = '/your-sitefinity-page',
                        failureCallback: (err) => {
                            document.querySelector('.msg').textContent =
                                'Login failed: ' + err;
                        }
                    });
                }
            });
        });
    </script>

Teams Auth Controller (Token Exchange)

This controller receives the Teams token, performs the On-Behalf-Of flow to get a Graph token, retrieves the user profile, and signs the user into Sitefinity.

Controllers/TeamsAuthController.cs
using Microsoft.AspNetCore.Authentication;
using Microsoft.AspNetCore.Authentication.Cookies;
using Microsoft.Identity.Client;
using System.Security.Claims;
using System.Net.Http.Headers;

[ApiController]
public class TeamsAuthController : ControllerBase
{
    private readonly IConfiguration _config;
    private readonly IHttpClientFactory _httpClientFactory;

    public TeamsAuthController(IConfiguration config, IHttpClientFactory httpClientFactory)
    {
        _config = config;
        _httpClientFactory = httpClientFactory;
    }

    // ── Main endpoint called from the Teams tab page ───────────────────────
    [HttpPost("/auth/teams-exchange")]
    public async Task<IActionResult> Exchange([FromBody] TokenRequest request)
    {
        try
        {
            var userInfo = await GetUserFromToken(request.Token);

            if (userInfo == null)
                return Ok(new { success = false, error = "Could not validate token" });

            // Build claims — same structure as your OpenID claims
            var claims = new List<Claim>
            {
                new Claim(ClaimTypes.Email,           userInfo.Email),
                new Claim(ClaimTypes.Name,            userInfo.DisplayName),
                new Claim(ClaimTypes.NameIdentifier,  userInfo.ObjectId),
                new Claim("preferred_username",       userInfo.Email),
                // Add any additional claims your Sitefinity personalization uses
            };

            var identity  = new ClaimsIdentity(claims, CookieAuthenticationDefaults.AuthenticationScheme);
            var principal = new ClaimsPrincipal(identity);

            // Sign in — creates the auth cookie Sitefinity Renderer reads
            await HttpContext.SignInAsync(
                CookieAuthenticationDefaults.AuthenticationScheme,
                principal,
                new AuthenticationProperties
                {
                    IsPersistent = true,
                    ExpiresUtc   = DateTimeOffset.UtcNow.AddHours(8)
                }
            );

            var redirectUrl = _config["TeamsTab:RedirectUrl"] ?? "/";
            return Ok(new { success = true, redirectUrl });
        }
        catch (Exception ex)
        {
            return Ok(new { success = false, error = ex.Message });
        }
    }

    // ── On-Behalf-Of flow: Teams token → Graph token → user profile ────────
    private async Task<AzureUserInfo?> GetUserFromToken(string teamsToken)
    {
        var tenantId     = _config["AzureAd:TenantId"];
        var clientId     = _config["AzureAd:ClientId"];
        var clientSecret = _config["AzureAd:ClientSecret"];
        var instance     = _config["AzureAd:Instance"];

        // Step 1: Exchange Teams token for a Graph-scoped token (OBO flow)
        var app = ConfidentialClientApplicationBuilder
            .Create(clientId)
            .WithClientSecret(clientSecret)
            .WithAuthority($"{instance}{tenantId}")
            .Build();

        var result = await app
            .AcquireTokenOnBehalfOf(
                new[] { "https://graph.microsoft.com/User.Read" },
                new UserAssertion(teamsToken)
            )
            .ExecuteAsync();

        // Step 2: Call Microsoft Graph /me with the new token
        var httpClient = _httpClientFactory.CreateClient();
        httpClient.DefaultRequestHeaders.Authorization =
            new AuthenticationHeaderValue("Bearer", result.AccessToken);

        var response = await httpClient.GetAsync("https://graph.microsoft.com/v1.0/me");

        if (!response.IsSuccessStatusCode)
            return null;

        var json      = await response.Content.ReadAsStringAsync();
        var graphUser = System.Text.Json.JsonSerializer.Deserialize<GraphUserResponse>(json);

        return new AzureUserInfo
        {
            Email       = graphUser?.Mail ?? graphUser?.UserPrincipalName ?? "",
            DisplayName = graphUser?.DisplayName ?? "",
            ObjectId    = graphUser?.Id ?? ""
        };
    }
}

// ── Models ──────────────────────────────────────────────────────────────────

public class TokenRequest
{
    public string Token { get; set; } = "";
}

public class AzureUserInfo
{
    public string Email       { get; set; } = "";
    public string DisplayName { get; set; } = "";
    public string ObjectId    { get; set; } = "";
}

public class GraphUserResponse
{
    [System.Text.Json.Serialization.JsonPropertyName("id")]
    public string? Id { get; set; }

    [System.Text.Json.Serialization.JsonPropertyName("displayName")]
    public string? DisplayName { get; set; }

    [System.Text.Json.Serialization.JsonPropertyName("mail")]
    public string? Mail { get; set; }

    [System.Text.Json.Serialization.JsonPropertyName("userPrincipalName")]
    public string? UserPrincipalName { get; set; }
}

Common Failure Points

# Problem Fix
1 webApplicationInfo missing or wrong in manifest Verify id matches your Azure Client ID and resource matches your App ID URI exactly
2 Cookie blocked in Teams iframe Ensure cookie has SameSite=None; Secure — both must be set
3 Page not loading in Teams (blank iframe) Remove X-Frame-Options header; add CSP frame-ancestors for Teams domains
4 Client secret error / 401 from Azure Check the secret hasn't expired in Azure AD; regenerate if needed
5 Authorized client app IDs missing Add both Teams client IDs under Expose an API → Authorized client applications
6 Admin consent not granted Go to API Permissions → click Grant admin consent
7 getAuthToken() fails with consent error The scope access_as_user must be added in Expose an API; Teams client IDs must be authorized
8 On-Behalf-Of flow fails (invalid_grant) Ensure the token audience matches your App ID URI; check tenant ID is correct
