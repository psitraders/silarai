1   How the Integration Works

Architecture overview — what talks to what

📲 WhatsApp Business API

  • Each merchant connects their own WhatsApp Business number
  • Uses Meta Cloud API (hosted by Meta, not on-premise)
  • Merchants get 1,000 free conversations/month from Meta
  • Connected via Embedded Signup (no manual API keys needed)

💬 Instagram & Facebook

  • Requires a Facebook Page linked to the business
  • Instagram DMs flow through the connected Facebook Page's token
  • Pro plan feature only (subscription gate in UI)
  • Merchant pastes Page ID + Page Access Token manually

2   Part A — Developer Setup

Do this ONCE as the platform owner. Merchants don't need to do any of these steps.

🛠

A1   Create a Meta App

Set up your Silarai app on the Meta Developer Portal

1

Go to Meta Developer Portal

Open developers.facebook.com and log in with your Facebook account (use the account tied to your business).

2

Create a New App

  • Click My Apps (top right) → Create App
  • Choose use case: "Other" → Next
  • Select app type: "Business" → Next
  • App name: Silarai (or your preferred name)
  • App contact email: your business email
  • Business account: select your Meta Business Account
  • Click Create App
3

Get Your App ID and App Secret

  • In your new app → Settings → Basic
  • Copy App ID (e.g. 1234567890123456)
  • Click "Show" next to App Secret and copy it
  • Save both — you'll need them in Step A4
Security: Never commit App Secret to GitHub. Store only in Azure App Service Configuration (see Step A4).
4

Set App Mode to Live (when ready for production)

  • Top of the App Dashboard → toggle from Development to Live
  • Keep in Development while testing — Live mode requires Meta App Review (Step A5)
📱

A2   Add Products to Your App

Enable WhatsApp, Messenger, and Instagram in your Meta App

5

Add WhatsApp

  • In App Dashboard sidebar → click Add Product
  • Find WhatsApp → click Set Up
  • Select or create a WhatsApp Business Account (WABA)
  • This creates a test phone number you can use immediately
6

Create WhatsApp Embedded Signup Configuration

  • In App sidebar → WhatsApp → Embedded Signup
  • Click Create Configuration
  • Name: Silarai Merchant Signup
  • Scopes to request: check whatsapp_business_management + whatsapp_business_messaging
  • Copy the Configuration ID (e.g. 1234567_embed_config) — needed for Step A4
7

Add Messenger

  • In App Dashboard → Add Product → Messenger → Set Up
  • Connect a Facebook Page if prompted (use your own page for testing)
8

Add Instagram Graph API

  • In App Dashboard → Add Product → Instagram Graph API → Set Up
  • Connect an Instagram Business account linked to a Facebook Page

A4   Add Azure App Service Environment Variables

Store Meta credentials securely in Azure — never in git

Important: Do NOT edit appsettings.json for these values — that file cannot be committed to GitHub (contains secrets). Use Azure App Service Configuration instead.
16

Open Azure App Service Configuration

  • Go to portal.azure.com
  • Navigate to your App Service: silarai-fbahb2bsg4cng3hq
  • Left sidebar → Settings → Configuration
  • Click the Application settings tab
17

Add the Following Environment Variables

Click + New application setting for each row below:

Name (Key) Value Where to Find
Facebook__AppId Your Meta App ID Meta Developer App → Settings → Basic → App ID
Facebook__AppSecret Your Meta App Secret Meta Developer App → Settings → Basic → App Secret
Facebook__WhatsAppConfigId Embedded Signup Config ID Meta App → WhatsApp → Embedded Signup → your config ID
Facebook__WebhookVerifyToken Any secret string you choose Make it up — e.g. sil-verify-2026. Must match what you entered in webhook setup (Steps 10, 13, 15)
Double-underscore __ notation: Azure App Service uses __ to represent nested JSON keys. So Facebook__AppId maps to appsettings.json path Facebook → AppId.
18

Save and Restart

  • Click Save at the top of the Configuration page
  • Confirm the restart prompt — the App Service will restart and pick up new values
  • Wait ~60 seconds for the restart to complete

A5   Submit App for Meta Review

Required before any merchant outside your own account can connect

Development mode allows: Testing with your own Facebook/WhatsApp accounts and any accounts you add as Test Users in the app. This is enough to test the full flow before review.
19

Prepare for Review

PermissionUsed For

whatsapp_business_messagingSending/receiving WhatsApp messages
pages_messagingSending/receiving Facebook Messenger messages
instagram_manage_messagesSending/receiving Instagram DMs
business_managementReading business info during Embedded Signup
20

Complete the Review Submission

  • Each permission requires a screencast video showing how it's used in your app
  • Submit a clear description: "Silarai allows merchants to connect their own WhatsApp Business number and receive customer messages as leads in their dashboard."
  • Make sure your app has a Privacy Policy URL and Terms of Service URL (required)
  • Typical review time: 3–7 business days

B1   WhatsApp via Embedded Signup

Automatic — available on all plans, takes ~3 minutes

This flow is fully automated. Merchants don't need to create a Meta Developer account or copy any API keys. Silarai captures everything automatically through Meta's Embedded Signup popup.
1

Go to Integrations Page

Log in to Silarai dashboard → Settings → Integrations

2

Click "Connect with Facebook"

  • The blue Facebook button appears under the WhatsApp Business card
  • If the button shows "Loading Facebook SDK…" wait 2–3 seconds for it to become active
3

Complete Meta's Embedded Signup Popup

  • Log in with a Facebook account that has access to your WhatsApp Business Account
  • Select your WhatsApp Business Account (WABA) — or create a new one
  • Select or register your WhatsApp phone number
  • Grant the requested permissions when prompted
  • Click Continue through all steps until the popup closes automatically
Don't have a WhatsApp Business number yet? Meta will let you register a new number during the signup flow. You can use an existing phone number or Meta provides a free test number.
4

Silarai Auto-Captures Your Credentials

  • When the popup closes, Silarai automatically receives: Phone Number ID, WABA ID, and Access Token
  • These are stored securely in the database, linked to your tenant account
  • The WhatsApp card will show a green Connected badge with your number
5

Send a Test Message (Optional)

  • Enter your phone number in the "Send test message" field (format: 919876543210 — country code, no +)
  • Click Test — you should receive a WhatsApp message within seconds
Troubleshooting Embedded Signup: If the popup closes but the card doesn't show "Connected", try the Manual Fallback option (click "Having trouble? Enter your IDs manually" below the Connect button). Get your Phone Number ID from Meta Developer App → WhatsApp → API Setup.

B2   Instagram DMs

Manual setup — Pro plan only

Prerequisite: You need a Facebook Page connected to your Instagram Business account. Personal Instagram accounts do not support the Messaging API.
6

Get Your Instagram Account ID

  • Go to business.facebook.com → Settings → Instagram Accounts
  • Your Instagram Business Account ID is shown there
  • Alternatively: Graph API Explorer → run GET /me?fields=instagram_business_account
7

Generate a Never-Expiring Page Access Token

  • Go to business.facebook.com → Settings → Page Access Tokens
  • Select your Facebook Page (the one connected to your Instagram)
  • Click Generate Token — choose Never expire
  • Copy the token (starts with EAA...)
Important: Instagram messaging uses your Facebook Page's access token — not your Instagram token. This is because Instagram DMs are processed through the connected Facebook Page.
8

Enter Credentials in Silarai

  • Silarai dashboard → Settings → Integrations → Instagram section
  • Paste your Instagram Account ID in the first field
  • Paste your Page Access Token in the second field
  • Click Save All Settings

B3   Facebook Messenger

Manual setup — Pro plan only

9

Get Your Facebook Page ID

  • Go to your Facebook Page
  • Click About → scroll to the bottom → find Page ID
  • Or: go to the page URL and click on "About" — the Page ID is in the URL for some pages
10

Generate a Page Access Token

  • Go to business.facebook.com → Settings → Page Access Tokens
  • Select your Facebook Page → Generate Token → Never expire
  • Make sure the token has pages_messaging permission
  • Copy the token (starts with EAA...)
11

Subscribe Your Page to the App

  • In Meta Developer App → Messenger → Settings
  • Under Access Tokens → Add or Remove Pages
  • Select the Facebook Page and grant permissions
  • This links your Page to receive webhook events
12

Enter Credentials in Silarai

  • Silarai dashboard → Settings → Integrations → Facebook Messenger section
  • Paste your Facebook Page ID
  • Paste your Page Access Token
  • Click Save All Settings

Test WhatsApp

13

Send a Test Message from the Dashboard

  • Integrations page → WhatsApp card → "Send test message"
  • Enter your phone number in format 919876543210 (country code + number, no +)
  • Click Test — you should receive a WhatsApp message in seconds
  • If it fails: check Azure App Service logs for error details
14

Send a Message to Your WhatsApp Number (Inbound Test)

  • Send any WhatsApp message TO the connected business number
  • Within seconds, a new Lead should appear in Silarai → Leads
  • The lead will show the sender's name and message preview

Test Webhook with Meta's Webhook Tester

15

Use Meta's Built-in Webhook Tester

  • In Meta Developer App → your app → WhatsApp → Configuration → Webhooks section
  • Click Test next to messages field
  • Meta sends a sample message payload to your webhook
  • Check Azure App Service logs: App Service → Monitoring → Log Stream — you should see the request come in
16

Verify Webhook Endpoint is Accessible

Open these URLs in a browser — they should return JSON (not an error page):

GET https://silarai-fbahb2bsg4cng3hq.southindia-01.azurewebsites.net/api/v1/webhook/whatsapp?hub.mode=subscribe&hub.verify_token=YOUR_TOKEN&hub.challenge=test123

Expected response: test123 (the challenge echoed back)

5   Troubleshooting

Common issues and how to resolve them

Issue Likely Cause Fix
Webhook verification fails
"Challenge not received"		 App Service is down, or Verify Token mismatch 1. Check App Service is running
2. Ensure Facebook__WebhookVerifyToken in Azure matches what you entered in Meta
3. Test URL manually in browser
Embedded Signup popup closes, no "Connected" status phoneNumberId not captured from Meta events Use the Manual Fallback: copy Phone Number ID from Meta Developer App → WhatsApp → API Setup and paste directly
WhatsApp messages sent but not received by customer Access token expired or Phone Number not registered Disconnect and reconnect WhatsApp via Embedded Signup to get a fresh token
Incoming messages not creating leads Webhook subscription missing or App in Development mode 1. Check webhook fields are subscribed in Meta app
2. In Development mode, only your own FB account generates real webhooks — add others as Test Users
Instagram DM webhook not firing Instagram not linked to Facebook Page, or wrong token 1. Ensure Instagram Business Account is linked to the Facebook Page
2. Regenerate the Page Access Token
3. Re-subscribe to Instagram webhooks in Meta app
"App not allowed" error during Embedded Signup App in Development mode + user not added as tester Add the merchant's Facebook account as a Tester in Meta App → Roles → Testers (Development mode only)
Test message button shows "Failed" Backend missing Facebook__AppId env var, or token invalid Check Azure App Service Configuration has all Facebook__* variables set and saved
Message sent twice / duplicate leads Multiple webhook subscriptions on same phone number Go to Meta Developer App → WhatsApp → Configuration → remove duplicate subscriptions
How to view backend logs: Azure Portal → App Service silarai-fbahb2bsg4cng3hq → Monitoring → Log Stream — real-time logs showing all webhook requests, errors, and responses.

  • Under the WhatsApp card → click "Webhook configuration (for Meta Developer App)" accordion
